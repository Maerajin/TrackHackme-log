## 실습 주제
(1) Introduction

(2) DNS:Remembering Addresses
 - 우리는 우리가 즐겨 사용하는 웹사이트의 IP주소를 일일이 외우지 않는다.
 - 우리가 이런 편의성을 누릴 수 있는 이유는 DNS(Domain name System)이라는 IP주소와 웹사이트의 이름을 연결해주는 프로토콜이 존재하기 때문이다.
 - DNS는 OSI7계층 중 응용계층에서 사용되고 DNS 트래픽은 UDP 53포트를 기본적으로 사용하고 폴백 시에 TCP 53포트를 사용한다.
 - DNS는 수많은 레코드를 가지고 있는데 이번 장에서는 그 중 다음 4가지만을 학습할 것이다.
   1) A record : 호스트이름을 하나 혹은 그 이상의 IPv4주소와 매핑하는 레코드이다.
   2) AAAA record : A record와 유사하지만, IPv6주소를 사용한다는 것에 차이점이 존재한다. AAAA는 순서대로 각각 Address, Authentication, Authorization, Accounting을 의미한다.
   3) CNAME record : 도메인 이름을 다른 도메인 이름과 연결하는 역할을 한다.
   4) MX record : 도메인을 관리하는 메일 서버를 관리하는 역할을 한다.

(3) WHOIS
 - 이렇게 도메인 이름이 IP주소로 매핑되어 표현되어진다면 어떤 레코드를 사용할건지 부여하는 권한이 필요하다.
 - WHOIS 프로토콜은 이러한 권한을 도메인 이름에 부여하는 프로토콜이다. 
 - 도메인 이름을 사용하기 위해서는 연간 비용을 내고 사용해야 한다. 하지만 타인에게 이러한 지불 정보를 비롯한 사적인 정보를 공개하고 싶지 않을 때 WHOIS 레코드를 사용하여 그러한 정보를 감춘다.
 - [](images/(3).png)

(4) HTTP(S) : Accessing the Web
 - 우리가 웹서핑을 위해 브라우저를 사용하면 HTTP(Hypertext Transfer Protocol), HTTPS(HTTP for Secure)프로토콜이 사용된다.
 - 이 프로토콜들은 웹 브라우저와 웹 서버간의 의사소통 방법을 정의하고 TCP 연결 방식을 사용한다.
 - 웹 브라우저가 웹 서버에 보내는 명령이나 메소드 중 일부는 다음과 같다.
   1) GET : HTML파일이나 이미지 파일과 같은 데이터들을 웹서버로부터 획득한다.
   2) POST : 파일을 업로드하거나 형식을 제출하듯 새로운 데이터를 서버에 올리는 것을 허락하는 메소드이다.
   3) PUT : 서버에 새로운 정보를 생성할 때나 기존의 정보를 업데이트하거나 덮어씌울 때 사용하는 메소드이다.
   4) DELETE : 서버에 있는 특정 파일이나 정보를 삭제할 때 사용하는 메소드이다.
 - HTTP, HTTPS는 주로 TCP 80, TCP 443번 포트를 사용하고, 그외에도 가끔 8080이나 8443번 포트도 사용한다.

(5) FTP : Transfering Files
 - 웹페이지의 정보를 가져오는 HTTP프로토콜과 다르게 FTP(File Transfer Protocol)은 이름 그대로 파일을 전송할 때 사용한다.
 - FTP는 파일 전송에 아주 효율적이고 동등한 상태라면 HTTP보다 더 빠른 속도를 낼 수 있다.
 - FTP에서 정의하는 명령어의 예는 다음과 같다.
   1) USER : 사용자 이름을 입력할 때 사용한다.
   2) PASS : 비밀번호를 입력하는데 사용한다.
   3) RETR : FTP서버로부터 사용자에게로 파일을 다운로드할 떄 사용한다.
   4) STOR : 클라이언트로부터 FTP서버로 파일을 업로드할 때 사용한다.
 - FTP서버는 TCP 21번 포트를 기본적으로 사용하고 데이터 전송은 다른 연결을 사용하여 수행한다.
 - [](images/(4).png)
   

(6) SMTP : Sending Email
 - SMTP(Simple Mail Transfer Protocol)은 어떤 방식으로 사용자와 서버가 대화를 할지 어떤 방식으로 사용자들끼리 메일을 주고받을지를 정의하는 프로토콜이다.
 - 간단한 비유를 들자면 SMTP는 네가 우체국에 가서 소포를 전하는 방식을 떠올리면 된다. 우체국에 가서 직원에게 어디에 보낼건지 보내는 사람은 누구인지 같은 것을 네트워크 상에서 실행한다고 보면 된다.
 - 메일 사용자가 SMTP서버로 메일을 전송할 때 흔히 쓰는 명령어는 다음과 같다.
   1) HELO, EHLO : SMTP세션을 시작한다.
   2) MAIL FROM : 송신자의 이메일 주소를 특정한다.
   3) RCPT TO : 수신자의 이메일 주소를 특정한다.
   4) DATA : 사용자가 보내는 이메일 메시지의 내용을 나타낸다.
   5) . : 이메일 메시지가 종료되었다는 것을 나타낸다.
 - SMTP는 주로 TCP 25번 포트를 사용한다.
 - [](images/(5).png)

(7) POP3 : Receiving Email 
 - POP3(Post Office Protocol version 3)는 사용자가 메일 서버와 소통하고 이메일 메시지를 받을 수 있도록 하는 프로토콜이다.
 - 기술적으로 깊이 들어갈 필요 없이, 사용자는 SMTP로 이메일을 보내고 POP3로 이메일을 받는다. SMTP는 우체국이고 POP3는 가정집에 있는 우체통이다.
 - POP3에서 주로 사용하는 명령어들은 다음과 같다.
   1) USER <username> : 사용자를 특정한다.
   2) PASS <password> : 사용자 비밀번호를 공급한다.
   3) STAT : 메세지의 수와 크기 정보를 요청한다.
   4) LIST : 모든 메시지와 메시지 크기를 목록으로 보여준다.
   5) RETR <message_number> : 특정 메시지를 받는다.
   6) DELE <message_number> : 특정 메시지의 삭제 마크를 남긴다.
   7) QUIT : 삭제와 같은 변경사항을 적용한 뒤 POP3세션을 종료한다.
 - POP3프로토콜은 기본적으로 TCP 110번 포트를 사용한다.

## 실무 적용 가능성
 - 오늘 베운 내용들(DHCP, NAT, ARP 등)은 이론적인 부분들이 많았다. 그래서 실무에서 직접적으로 사용할 일은 없을 것 같다. 생각해보자면 보고서를 작성하는 정도가 아닐까 생각해본다.
 - 내가 가고 싶은 진로인 보안관제(SOC)에서는 사용할 일이 없더라도 알아야 하는 내용이니 외우는 것이 좋다고 생각한다.
